# 📦 Distribution System
An end-to-end, online-first business management and distribution tracking platform designed specifically for mobile field workers and central administrators.

The system is engineered into three interconnected tiers:
1. **Offline-First Mobile App** (React Native / Expo)
2. **Central Admin Web Panel** (Next.js 14 / TailwindCSS)
3. **Synchronization REST API** (Python / FastAPI / SQLite)

---

## 🌟 Key Features

### 1. 📱 Field Worker Mobile App (Offline-First)
Designed for low-end Android devices in areas with unstable internet connections, allowing the workforce to operate fully offline and sync later.
* **Offline SQLite Database**: Caches all shop lists, product catalogs, and pricing data locally on the device.
* **Role-Based Workflows**:
  * **Order Takers**: Record shop orders detailing exact Carton/Piece quantities.
  * **Salesmen**: Manage dispatch realization, recording exact cash income received vs. outstanding loan (credit) given to shops.
* **Smart Unit Handling**: Supports "Carton" and "Piece" transactions dynamically.
* **Background Auto-Sync**: Accumulates transactions locally and automatically synchronizes with the central server the moment an internet connection is detected.

### 2. 💻 Admin Web Panel
A comprehensive, responsive dashboard built with Next.js and styled perfectly with TailwindCSS for the core administration team.
* **Staff & User Management**: Create and manage accounts with specific access roles (`admin`, `ordertaker`, `salesman`).
* **Customer Network (Shops)**: Manage the distribution network, adding and updating shop details.
* **Dynamic Inventory & Dual-Pricing System**:
  * Complete product catalog management.
  * Define unique unit types and set specific prices for both "Cartons" and "Pieces" per item.
* **Live Activity Logs**:
  * **Order Logs**: Track exact dispatch orders sent from Order Takers in the field.
  * **Sales Records**: Monitor exactly how much cash was received vs. outstanding shop credit generated by Salesmen.
* **Timezone Perfection**: Enforces strict `Pakistan Standard Time (PKT / UTC+5)` across the entire system, normalizing naive local datetimes seamlessly.
* **Advanced Deletion Controls**: Admins can permanently delete individual erroneous logs (trash-icon per row) or bulk-delete entirely by specific calendar dates.

### 3. 🖨️ Professional Reporting System
* **Dispatch Checklists**: Generate beautiful, physical printout sheets for "Orders Dispatch Summary" meant to be handed to delivery drivers.
* **Revenue Realizations**: Print highly detailed "Salesman Revenue Realization" ledger sheets calculating total exact cash inflow vs. credit outflow per day.
* **Professional Print Branding**: The web app dynamically hides all default browser headers/footers (e.g., `localhost` or URL tags), injecting clean custom footers that read: **"DEVELOPED BY NAVEED DEVELOPER"** and printing top titles as **"Mohammad Naveed Mostafa"**.
* **Excel / CSV Exports**: 1-click export of structured `.csv` datasets for accounting software or Excel processing.

## 🚀 Getting Started

### 1. Backend (FastAPI) Setup
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
*The API will be available at `http://localhost:8000` (or your local IP for mobile connections).*

### 2. Admin Panel (Next.js) Setup
```bash
cd admin-panel
npm install
npm run dev
```
*The web dashboard will be available at `http://localhost:3000`.*

### 3. Mobile App (Expo) Setup
Before starting, ensure the `API_URL` inside `mobile-app/config.js` points to your active backend IP address (e.g., `http://192.168.1.XX:8000`).
```bash
cd mobile-app
npm install
npx expo start -c
```
*Use the Expo Go app on your Android device or an Android Emulator to launch the application.*

## 🔒 Security & Data Integrity
* Role-based access control prevents Salesmen from entering Order Taker routes and vice-versa.
* Bulletproof API date-filtering prevents double-deletions.
* Synchronize-tracking logic ensures that pending local database entries do not duplicate on the central server during network dropouts.
